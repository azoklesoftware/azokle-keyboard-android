# Security & Threat Model

Azokle Keyboard relies on the vetted Double Ratchet Algorithm and X3DH implementations mapped over to standalone channels. However, if you discover a vulnerability either within our envelope structures, key rotation schedules, or localized Android memory configurations, we urge you to report it directly so we can patch the underlying protocols safely.

## Reporting a Vulnerability

**Do not publicly disclose vulnerabilities** on GitHub issues or other public forums. Please responsibly disclose zero-days, cryptographic misconfigurations, or memory-leak crashes to our security operations team.

📧 **Contact:** [security@azokle.com](mailto:security@azokle.com)

Please provide a detailed Proof of Concept (PoC). We actively prioritize disclosures and aim for a 48-hour response lifecycle.
